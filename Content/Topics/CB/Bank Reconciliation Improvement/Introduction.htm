Bank Reconciliation Improvements (24770) - Introduction

Changes have been made to address a limitation in OA that prevented more than 100 transactions being tagged and actioned on a browser page. This could cause a problem in CB bank reconciliation when trying to match more than 100 items to a bank statement detail record.

This new functionality allows up to 100 items at a time to be ‘marked’ for reconciliation against a statement item that may cover any number of items. The user can then repeat the process until all the items that are to be reconciled to the statement item have been marked.

A new action on the Statement Items page allows the user to reconcile the statement item to the marked items.

This also has the benefit of not having to tag all items at the same time to perform reconciliation. Thus the user can mark items at different times as well as update the marked items easily.
